# Canonical public moment pages — Phase 1 MVP

## Summary

Shipped **`/moment/[id]/[slug]`** as the crawlable, canonical surface for a **materialized** subset of transcript moments. Stable **20-hex** ids are derived from `(videoId, startSeconds, phrase)` via SHA-256. Content is backed by **`data/public-moments.json`** (generated by **`npm run materialize:public-moments`**, same transcript cache + `suggestKeywords` + `hybridFindMatches` pipeline as the legacy moment sitemap).

## What was implemented

| Area | Detail |
|------|--------|
| Route | `app/moment/[id]/[slug]/page.tsx` — **ISR `revalidate = 3600`**, `generateStaticParams` from JSON, **308-style slug correction** via `permanentRedirect` when slug ≠ `canonicalSlug`, `notFound` for unknown / inconsistent ids |
| Metadata | `createPublicMomentMetadata` in `lib/seo.ts` — canonical URL, OG/Twitter large image, `robots: index, follow` |
| JSON-LD | `buildPublicMomentStructuredData` — `WebPage` + `VideoObject` + `BreadcrumbList` |
| OG image | `GET /api/og/moment-public/[id]` — reads materialized row + indexed video fallbacks |
| Transcript excerpt | Live `fetchTranscriptByVideoId` + `hybridFindMatches` when possible; falls back to materialized snippet |
| Save / share | `SaveMomentButton`, `ShareActions`, extended **`MomentSharePanel`** (`trackedMomentPageUrl`, `momentOgImageUrl`, `viralMomentPageUrl`) |
| Related moments | `getRelatedPublicMoments` — same video / phrase overlap / channel heuristic, then deterministic fill |
| Internal links | Video page, cross-video search, legacy moment URL, transcripts index, trending |
| Sitemap | `loadPublicMoments()` merged in `app/sitemap.ts` (no extra env flag; empty file adds nothing) |
| Analytics | `canonical_moment_page_view`, `canonical_moment_youtube_click`, `canonical_moment_related_click` |

## Operational notes

- **Regenerate materialized JSON** after meaningful cache/index changes: `npm run materialize:public-moments` (`PUBLIC_MOMENTS_CAP`, `PUBLIC_MOMENTS_PER_VIDEO` optional).
- **Slug stability** is guaranteed for committed rows; changing `phrase`/`startSeconds` for the same id breaks `isPublicMomentRecordConsistent` → **404** (intentional guard).

## Verification (this branch)

- **`npm run lint`** — pass  
- **`npm run build`** — pass (267 static paths including `/moment/...`)  
- **`npm run audit:seo:quick`** — run against local `next start` hit **Next.js / Turbopack client-manifest `InvariantError` and 500s** on several routes (including `/moment/...`), consistent with other dynamic/app routes in the same session — **not treated as a moment-page HTML regression**; re-run audit against a healthy deployment or `next dev` if local `next start` is unstable.

## Residual risks

1. **Materialization drift** — If transcripts or hybrid ranking shift, live excerpt can differ from OG card (still both transcript-backed).  
2. **Id collision** — Truncated 20-hex id is extremely unlikely but theoretically possible; full hash could be used later if needed.  
3. **JSON edit errors** — Manual edits that break id consistency surface as **404**.  
4. **Build / NFT tracing** — `readFileSync` for `data/public-moments.json` may contribute to Turbopack “unexpected file in NFT list” warnings alongside existing `process.cwd()` data access patterns.  
5. **SEO audit local** — `next start` + quick audit may be flaky depending on Next 16.2.x + Turbopack output; prefer CI or production smoke for audit gates.
